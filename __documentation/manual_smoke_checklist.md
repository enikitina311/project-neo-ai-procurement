# Procurement Service Manual Smoke Checklist

## Scope
Чек-лист для ручной проверки сервиса закупки NEO после внутреннего рефакторинга backend/frontend без изменения публичных контрактов.

## Environment
- Backend plugin: `project-neo-be-procurement`
- Frontend: `project-neo-fe-dev`
- AI integrations: `n8n` webhooks procurement flows
- OCR: `ProcurementOcrService`

## Core Flow
1. Открыть страницу сервиса закупки и убедиться, что список пакетов загружается без ошибок.
2. Создать новый пакет закупки.
3. Проверить, что после создания открывается карточка нового пакета.
4. Изменить название, критерии, `coverageThreshold`, `suppliersLimit` и сохранить пакет.
5. Обновить страницу и убедиться, что данные пакета сохранились.

## Items
1. Добавить позицию номенклатуры через строку добавления.
2. Добавить несколько позиций подряд.
3. Изменить `name`, `specs`, `qty`, `unit` у существующей позиции.
4. Удалить одну из позиций.
5. Убедиться, что список позиций корректно обновляется без перезагрузки страницы.

## Supplier Search
1. Запустить поиск поставщиков по пакету.
2. Проверить, что при долгом запросе виден loading state.
3. Проверить отображение списка поставщиков из `n8n`.
4. Проверить фильтрацию по `coverageThreshold`.
5. Выбрать несколько найденных поставщиков и перенести их в пакет.
6. Убедиться, что дубликаты по имени не создаются при повторном переносе.
7. Переключить `selected` у поставщика в списке текущих поставщиков.

## Letters
1. С выбранными поставщиками запустить генерацию писем.
2. Проверить, что виден loading state генерации.
3. Проверить, что на вкладке появились новые черновики писем.
4. Снять выбор со всех поставщиков и повторить генерацию.
5. Убедиться, что сервис корректно fallback-ится на всех поставщиков пакета.
6. Повторно запустить генерацию и проверить, что старые draft-записи заменяются новыми.

## KP / OCR / Analysis
1. Выбрать поставщика на вкладке КП.
2. Загрузить PDF или изображение коммерческого предложения.
3. Проверить, что upload button блокируется без выбранного поставщика и без файла.
4. Убедиться, что после загрузки документ появляется в таблице.
5. Запустить анализ КП.
6. Проверить отображение `isComplete`, `missingFields`, `notes`.
7. Обновить страницу и убедиться, что сохраненный анализ повторно загружается.

## NMC
1. При наличии проанализированных КП запустить генерацию НМЦ.
2. Проверить loading state кнопки.
3. Проверить, что результат сохраняется и отображается после обновления страницы.

## Negative Cases
1. Остановить или сломать соответствующий `n8n` webhook и убедиться, что фронт показывает toast-ошибку, а UI остается рабочим.
2. Проверить ответ `n8n` в виде fenced JSON и убедиться, что supplier search / analysis / NMC продолжают работать.
3. Проверить ответ `n8n` с частично невалидным текстом и убедиться, что backend не падает необработанным исключением.
4. Удалить файл КП из file storage и убедиться, что backend возвращает контролируемую ошибку.
5. Спровоцировать OCR failure и убедиться, что UI получает ошибку без зависания.
